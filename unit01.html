Unit 01: 공간 보간 및 지구통계학 (Spatial Interpolation & Geostatistics)

🌏 1. 기본 개념 (공통 목표)

  • 📌 해결 과제: 캘리포니아 전역의 기상 관측소 온도 데이터만 가지고 있을 때, 관측소가 없는 나머지 모든 지역의 예상 온도를 수학적으로 추정해서 채워 넣는 작업입니다. 이를 통해 가장 덥거나 추운 도시를 찾아낼 수 있습니다.
  • 📌 필요한 데이터:
       - 점(Point) 데이터: 측정값이 포함된 기상 관측소 온도 CSV.
       - DEM(Digital Elevation Model): 지형의 고도가 온도에 미치는 영향을 이해하기 위한 데이터.
       - 경계 폴리곤(Polygon): 결과물을 집계할 도시 경계선 데이터.
  • 📌 핵심 공간 개념:
       - 토블러의 지리학 제1법칙: "모든 것은 관련되어 있지만, 가까운 것들이 먼 것들보다 더 비슷하다."
       - IDW (역거리 가중치): 단순히 거리에 반비례하여 빈 공간을 채우는 방식입니다.
       - Kriging (크리깅): 데이터의 통계적 패턴까지 고려하여 채우는 똑똑한 지구통계학적 방식입니다.

ArcGIS Pro 구현

도구 숙련자 및 오퍼레이터 양성

ArcGIS 워크플로우는 사용자가 직접 화면의 버튼을 누르고, 설정 값을 입력하여 통계 마법사를 실행하는 데 중점을 둡니다.

Step 1

데이터 획득 및 점 변환

NOAA 웹사이트에서 CSV 데이터를 다운로드하고, 'Display XY Data' 기능을 사용하여 숫자를 지도 위의 실제 점으로 변환합니다.

Step 2

지구통계 마법사 (Geostatistical Wizard)

복잡한 통계 계산을 도와주는 마법사 창을 통해 EBK(Empirical Bayesian Kriging) 모델을 선택합니다.

Step 3

구역 통계 (Zonal Statistics)

분석된 온도 지도를 바탕으로 각 도시 경계 내의 평균 온도를 계산하여 최종 테이블을 도출합니다.

Python 로직 및 아키텍처

지식 창조자 및 GIS 엔지니어 양성

파이썬 방식은 API를 통한 자동 수집과 수학적 알고리즘 직접 제어에 집중합니다. 이번 실행에서는 Meteostat에서 1991-2020 기후 정상값을 가져오고, PyKrige로 1월/8월 기온면을 만든 뒤, 도시 경계 자료와 겹쳐 PDF 리포트를 자동 생성했습니다.

Step 1

자동화된 데이터 수집 (API)

매번 웹사이트에 접속하는 대신 meteostat 라이브러리를 사용하여 코드가 기상청 서버에서 데이터를 긁어옵니다.

Step 2

알고리즘적 보간 (PyKrige)

마법사 창 대신 OrdinaryKriging 클래스를 호출합니다. 수학 공식을 코드에 직접 넣어 격자판의 숫자를 계산합니다.

Step 3

메모리 기반 구역 통계

결과를 파일로 저장하지 않고 컴퓨터 메모리 위에서 rasterstats를 사용하여 평균값을 즉시 뽑아냅니다.

[핵심 로직 2] 빈칸 채우기 마법 (Kriging 보간법)
# [설명] 날씨 지점이 없는 '빈 땅'의 온도가 궁금할 때 쓰는 수학적 방법입니다. # "가까운 곳끼리는 날씨가 비슷할 거야!"라는 원리를 이용해 캘리포니아 전체를 촘촘한 기온 지도로 채웁니다. from pykrige.ok import OrdinaryKriging # 1. 'OrdinaryKriging'이라는 빈칸 채우기 도구를 준비합니다. # 여기에 아까 모은 위경도(lon, lat)와 기온(temp)을 집어넣습니다. ok = OrdinaryKriging(df.lon, df.lat, df.temp, variogram_model='spherical') # 2. 캘리포니아 지도를 촘촘한 그물망(Grid)으로 나누고 전체 기온을 한 번에 계산(execute)합니다. # 이제 지점 데이터가 '면' 형태의 거대한 지도로 변신합니다! z_pred, ss = ok.execute('grid', grid_lon, grid_lat)
