Unit 03: 가시권 분석 (Viewshed Analysis)

🌏 1. 기본 개념 (공통 목표)

  • 📌 해결 과제: 울퉁불퉁한 지형 위에서 특정 관측 지점(셀 타워)에 섰을 때 눈으로 볼 수 있는 영역(가시권)을 파악합니다. 이를 통해 T-Mobile 통신 기지국의 무선 주파수가 지형에 막히지 않고 도달할 수 있는 커버리지 영역을 모델링합니다.
  • 📌 필요한 데이터:
       - 지형의 높낮이를 담은 고도 데이터 ( DEM : 1-arc-second 고해상도 이미지 데이터).
       - 관측 지점 데이터 (셀 타워의 위치 좌표, 타워 높이).
  • 📌 핵심 공간 개념:
       - 가시선 (LOS, Line of Sight): 타워 꼭대기에서 지면의 특정 픽셀까지 장애물 없이 이어지는 직선.
       - 매개변수 약속: 분석을 위해 타워의 높이, 송신 최대 반경(25km), 사람의 휴대전화 높이(지면에서 1.5m) 등 현실의 물리법칙을 데이터화.

ArcGIS Pro 구현

도구 숙련자 및 오퍼레이터 양성

ArcGIS 워크플로우는 래스터 데이터를 처리하기 위한 전용 유료 도구 상자인 Spatial Analyst 익스텐션을 활용합니다. 아래는 개념적인 작업 흐름입니다.

개념 1

래스터 준비 (모자이크 처리)

여러 장의 DEM(지형 고도 데이터) 타일을 하나의 거대한 지형 이미지로 합칩니다. 이때 각 파일의 좌표계(Spatial Reference)가 동일한지 일일이 확인해야 오류가 나지 않습니다.

개념 2

속성 테이블 조작 (필드 매핑)

가시권 분석 도구가 인식할 수 있도록 약속된 이름(하드코딩된 필드명)으로 데이터 표를 수정해야 합니다. 예를 들어 타워 높이는 무조건 OFFSETA 라는 이름의 기둥(Column)에 적혀 있어야 컴퓨터가 알아먹습니다.

개념 3

가시권 도구 실행

도구를 실행하면 컴퓨터가 DEM 이미지의 수많은 픽셀을 스캔하며 타워와 픽셀 사이의 시야를 가리는 장애물이 있는지 수학적으로 검사합니다.

Python 로직 및 아키텍처

지식 창조자 및 GIS 엔지니어 양성

파이썬 프로그래밍 방식은 고성능 래스터 처리를 위해 GDAL과 rasterio를 직접 다루며, 컴퓨터의 모든 두뇌(CPU 코어)를 동시에 사용하게 만드는 아키텍처 최적화에 집중합니다. 최종 PDF는 검증된 coverage_results.csv 값을 기준으로 다시 생성되어, HTML 차트와 PDF 표의 시나리오 수치가 일치합니다.

Step 1

프로그래밍 방식 래스터 처리

래스터(격자 이미지) 데이터를 읽고 쓰는 파이썬 도구 상자인 rasterio 라이브러리를 사용합니다. ArcGIS처럼 수동으로 도구를 실행해 파일을 저장하는 대신, 코드 안에서 rasterio.merge 라는 함수(명령어)를 호출하여 여러 장의 DEM 데이터 배열(Array)을 컴퓨터 메모리 위에서 즉시 동적으로 이어 붙입니다.

Step 2

알고리즘적 가시권 계산

ArcGIS의 무거운 UI 창을 통하지 않고, 파이썬 코드에서 gdal 라이브러리 모듈을 불러온 뒤 ViewshedGenerate 라는 함수(가시권을 계산하는 핵심 로직)를 직접 실행합니다. 속성 테이블에 이름을 바꿔 적는 번거로운 짓을 할 필요가 없습니다. 코드에서 이 함수에 observer_height=30 (관측자 높이 30미터), target_height=1.5 처럼 매개변수(인자 값)를 명시적으로 던져주기만 하면 됩니다.

Step 3

시스템 아키텍처 (병렬 처리)

해상도가 매우 높은 DEM 지형 위에서 수백 개의 셀 타워를 한 번에 하나씩 검사하는 것은 엄청나게 느립니다. 파이썬에 내장된 concurrent.futures 모듈 안에서 ProcessPoolExecutor 라는 클래스(작업 반장 역할을 하는 객체 틀)를 불러옵니다. 이 작업 반장에게 지시하여, 거대한 계산 작업들을 여러 개의 뭉치로 쪼갠 뒤 컴퓨터에 있는 여러 개의 CPU 코어(예: 32개 코어)에 골고루 던져주어 "동시에" 계산하게 만듭니다.

※ 지식 창조자의 무기: "클릭" 대신 컴퓨터의 두뇌와 직접 소통하는 방법. 파이썬 문법을 몰라도, 회색으로 적힌 한글 해설을 읽어보며 코드가 어떻게 흐르는지 감을 잡아보세요!

[핵심 로직 0] 여러 장의 지형 래스터 데이터를 하나로 합치기 (모자이크)

ArcGIS에서 마우스로 파일들을 하나씩 골라서 도구를 실행했던 그 작업을, 파이썬에서는 rasterio 라이브러리를 통해 동적으로 메모리 위에서 이어 붙입니다.

# 여러 장의 DEM 파일을 담아둘 빈 리스트를 준비합니다. src_files_to_mosaic = [] # 폴더에 있는 파일(DEM_TILES)을 하나씩 꺼내면서 반복합니다. for fp in DEM_TILES: # rasterio 도구를 써서 파일을 열어줍니다. src = rasterio.open(fp) # 열린 파일을 리스트 바구니에 차곡차곡 담습니다. src_files_to_mosaic.append(src) # 🌟 가장 중요한 마법의 함수! merge 함수가 바구니에 담긴 조각들을 하나의 거대한 지도로 알아서 꿰매줍니다. # mosaic에는 하나로 합쳐진 거대한 데이터 배열(Array)이, out_trans에는 좌표 정보가 담깁니다. mosaic, out_trans = merge(src_files_to_mosaic)
[핵심 로직 1] 가시권(Viewshed) 계산 명령 내리기

ArcGIS에서 45분 걸리는 작업을 단 몇 초 만에 끝내기 위해, 컴퓨터 내부의 강력한 지리 연산 엔진(GDAL)을 직접 조종하는 코드입니다.

# --- 가시권(Viewshed) 분석을 수행하는 핵심 작업 반장 --- def run_single_viewshed(args): # args라는 꾸러미에서 필요한 정보(위치, 타워 높이 등)를 하나씩 꺼냅니다. idx, x, y, tower_height, radius, dem_path, out_path = args # gdal이라는 라이브러리(강력한 공간 데이터 도구상자)를 이용해 지형 고도 데이터(DEM)를 엽니다. src_ds = gdal.Open(dem_path) # 고도 데이터의 첫 번째 껍질(Band)을 가져옵니다. (여기에 실제 산과 계곡의 높이 숫자들이 빼곡히 들어있어요) band = src_ds.GetRasterBand(1) # 관측자(타워)의 높이는 아까 꺼낸 'tower_height' 숫자를 씁니다. observer_height = tower_height # 목표물(사람)의 핸드폰 높이는 땅에서 1.5미터라고 정해줍니다. target_height = 1.5 # 🌟 가장 중요한 순간! GDAL 엔진에게 가시권(Viewshed)을 계산하라고 직접 명령합니다. gdal.ViewshedGenerate( band, "GTiff", out_path, ["COMPRESS=LZW"], # 어떤 그림 파일로 예쁘게 압축해서 저장할지 설정 x, y, observer_height, target_height, # 타워의 위치(x,y)와 관측자/목표물 높이 정보 전달 1, 0, 0, -9999, # 보이는 곳은 1, 안 보이는 곳은 0으로 색칠하라는 수학적 약속 0.85714, 2, radius # 지구는 둥그니까 곡률을 계산에 넣고, 최대 탐색 거리(radius)를 알려줌 ) # 계산이 끝났으니 열어두었던 지형 데이터 파일을 닫습니다. (컴퓨터 메모리 정리정돈) src_ds = None # 완성된 결과 파일의 이름을 메인 프로그램으로 반환(보고)합니다. return out_path
[핵심 로직 2] 수백 개의 타워를 32개 코어로 동시(병렬) 처리하기

파이썬의 진정한 마법입니다. 한 번에 하나씩 1시간 넘게 걸려서 계산하지 않고, 컴퓨터의 모든 두뇌를 동시에 깨워서 일하게 만듭니다.

# 빈 리스트(결과 파일 이름들을 담아둘 빈 바구니)를 만듭니다. generated_tifs = [] # 컴퓨터에 있는 모든 두뇌(CPU 코어, 예를 들어 32개)를 전부 동원하는 총감독(ProcessPoolExecutor)을 부릅니다! with ProcessPoolExecutor(max_workers=os.cpu_count()) as executor: # 수백 개의 타워 위치(tasks)를 32개의 두뇌에게 골고루 나눠주고 동시에 '위의 가시권 계산'을 시작(map)합니다! for res in executor.map(run_single_viewshed, tasks): # 두뇌들이 각자 계산을 마칠 때마다, 완성된 결과 파일 이름을 바구니에 하나씩 차곡차곡 담습니다. generated_tifs.append(res)
