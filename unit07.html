Unit 07: ModelBuilder 해수면 상승 도구

1. 과제 정의

  • 해결 과제: DEM raster를 입력받아 사용자가 지정한 sea level change와 area of concern radius를 반영하는 ArcGIS ModelBuilder tool을 만듭니다.
  • 제출물: tool이 들어 있는 toolbox file과, ModelBuilder workflow 및 tool interface screenshots가 들어간 PDF document 두 가지입니다.
  • 필수 입력값: input raster dataset, sea level change in feet, area of concern radius in feet, output polygon path 네 항목을 모두 tool parameter로 노출해야 합니다.
  • 필수 처리 순서: meters-to-feet Raster Calculator, 0/1 inundation Raster Calculator, Raster to Polygon, Make Feature Layer, Buffer 순서입니다.
  • 기본 테스트: 제공 DEM `unit07-010-n32_w081_1arc_v2.tif`, sea level change `6 ft`, area of concern radius `500 ft`로 실행했습니다.

ArcGIS Pro 구현

ModelBuilder tool 제작 매뉴얼

ArcGIS 방식의 핵심은 지도를 한 번 만드는 것이 아니라, 다른 DEM과 다른 sea-level 값을 넣어도 다시 실행되는 tool을 만드는 것입니다.

개념 1

ModelBuilder는 보이는 프로그램

도구 상자 안의 각 도구를 선으로 연결하면 하나의 작은 프로그램이 됩니다. 이번 과제에서는 Raster Calculator부터 Buffer까지를 하나의 실행 흐름으로 묶습니다.

개념 2

Parameter는 사용자가 채우는 빈칸

input raster, sea level change, buffer radius, output path가 tool dialog에 보여야 합니다. 숨겨진 model variable이면 채점 기준의 parameter 점수를 잃습니다.

개념 3

PDF는 실행 증빙 문서

PDF에는 예쁜 설명보다 model canvas, tool interface, parameter defaults, output layer가 실제로 보이는 스크린샷이 중요합니다.

Python 구현

ModelBuilder 내부 로직 검증 파이프라인

Python은 ArcGIS 제출물을 대신하지 않습니다. 대신 같은 DEM과 같은 기본값으로 ModelBuilder가 어떤 계산을 하는지 눈에 보이는 PDF/지도/CSV로 검증합니다.

핵심 1

DEM은 칸마다 높이가 적힌 지도

13백만 개 가까운 DEM cell을 열고, 각 cell의 meter 값을 feet로 바꿉니다. 이 단계가 첫 번째 Raster Calculator와 같습니다.

핵심 2

0/1 스티커 지도 만들기

6 ft 기준보다 낮은 칸은 침수 후보로 표시합니다. 컴퓨터 입장에서는 파란 스티커와 빈 스티커를 전부 붙이는 일입니다.

핵심 3

붙어 있는 칸을 polygon으로 묶기

Raster to Polygon처럼 인접한 침수 cell들을 하나의 색칠 구역으로 묶고, 그 주변에 500 ft 안전띠를 둘러 final area of concern을 만듭니다.

Python 코어 로직 해설
[핵심 로직 0] DEM 파일 열기
# DATA_DIR은 Unit 07 원자료 폴더입니다. # 높이 숫자가 칸마다 적힌 DEM 파일을 실제로 엽니다. INPUT_DEM = DATA_DIR / "unit07-010-n32_w081_1arc_v2.tif" with rasterio.open(INPUT_DEM) as src: data_m = src.read(1, masked=True).astype("float32") data_ft = data_m * 3.28084
[핵심 로직 1] sea level 기준으로 0/1 mask 만들기
# sea_level_change_ft는 tool dialog에서 학생이 입력하는 숫자입니다. # 이번 기본 테스트에서는 6 ft를 넣었습니다. sea_level_change_ft = 6 valid = ~np.ma.getmaskarray(data_ft) # 기준보다 낮거나 같은 칸을 True로 표시합니다. # 이것이 ModelBuilder의 두 번째 Raster Calculator 결과입니다. inundated = valid & (data_ft <= sea_level_change_ft) new_scenario_band = valid & (data_ft > 0) & (data_ft <= sea_level_change_ft)
[핵심 로직 2] Raster to Polygon 재현
# 0/1 스티커 지도를 polygon 조각으로 바꿉니다. # 붙어 있는 침수 칸들이 하나의 색칠 구역으로 묶입니다. raster = inundated.astype("uint8") geometries = [ shape(geom) for geom, value in features.shapes(raster, mask=inundated, transform=transform) if int(value) == 1 ] inundated_polygons = gpd.GeoDataFrame(geometry=geometries, crs="EPSG:4326")
[핵심 로직 3] Make Feature Layer와 Buffer 재현
# ArcGIS의 gridcode = 0 선택과 같은 결과만 남긴 뒤, # 500 ft를 meter로 바꿔 projected CRS에서 buffer를 두릅니다. TARGET_CRS = "EPSG:32617" radius_m = 500 * 0.3048 projected = inundated_polygons.to_crs(TARGET_CRS) merged = unary_union(projected.geometry) area_of_concern = merged.buffer(radius_m)
[핵심 로직 4] PDF, PNG, CSV 저장
# 결과를 학생이 검토할 수 있는 산출물로 저장합니다. # PDF는 채점표 체크리스트, workflow 그림, 지도 2장을 포함합니다. summary.to_csv(DOC_DIR / "unit07_scenario_summary.csv", index=False) fig.savefig(DOC_DIR / "unit07_sea_level_scenario_map.png", dpi=180) build_pdf( DOC_DIR / "Sea_Level_Rise_ModelBuilder_UCLA_Unit07.pdf" )
