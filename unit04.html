Unit 04: 지도 자동화 및 대량 생산 (Map Series & Automation)

🌏 1. 기본 개념 (공통 목표)

  • 📌 해결 과제: CalEnviroScreen 4.0의 센서스 트랙 단위 환경·건강 지표를 사용해 캘리포니아 58개 카운티별 지도를 만듭니다. 선택한 변수는 2개이므로 최종 지도 페이지는 58개 카운티 × 2개 변수 = 116장입니다.
  • 📌 필요한 데이터:
       - 인덱스 레이어(Index Layer): 지도책의 페이지를 나누는 기준인 캘리포니아 58개 카운티 경계.
       - 주제 데이터: CalEnviroScreen 4.0 센서스 트랙 데이터. 이번 Python 산출물은 PolBurdP(Pollution Burden Percentile)와 AsthmaP(Asthma Percentile)를 사용했습니다.
       - 최종 결과물: 표지, 요약/자동화 논의, 116개 지도 페이지가 합쳐진 단일 PDF atlas.
  • 📌 핵심 공간 개념:
       - Map Series (지도 시리즈): 한 장짜리 지도 틀을 준비해 두면, 카메라가 카운티 경계를 하나씩 따라가며 같은 형식의 지도를 자동으로 찍어내는 기능입니다.
       - Fixed Legend (고정 범례): 모든 페이지에서 0-100 percentile 기준을 유지해야 카운티끼리 공정하게 비교할 수 있습니다.
       - Dynamic Text (동적 텍스트): 지도마다 카운티 이름, 변수 이름, 페이지 번호가 자동으로 바뀌는 글씨입니다.

ArcGIS Pro 구현

도구 숙련자 및 오퍼레이터 양성

ArcGIS 워크플로우는 레이아웃을 한 번 정교하게 만든 뒤, Spatial Map Series가 카운티 인덱스 레이어를 따라 자동으로 58페이지씩 찍어내도록 설정하는 과정입니다.

Step 1

지도와 레이아웃 준비

CalEnviroScreen 트랙 레이어와 카운티 경계 레이어를 올리고, 첫 번째 변수의 심볼을 0-100 percentile 고정 구간으로 설정합니다. 같은 범례가 모든 카운티 페이지에 유지되어야 합니다.

Step 2

Spatial Map Series 활성화

Layout view에서 Map Series를 켜고, 카운티 경계를 Index Layer로 지정합니다. 이 순간부터 카운티 하나가 PDF 한 페이지의 카메라 위치가 됩니다.

Step 3

동적 요소와 PDF 내보내기

Map Series용 page name, page number with count, 고정 legend를 배치합니다. 첫 번째 변수 58장을 내보낸 뒤 두 번째 변수에도 같은 틀을 적용해 총 116개 지도 페이지를 만듭니다.

Python 로직 및 아키텍처

지식 창조자 및 GIS 엔지니어 양성

파이썬 방식은 ArcGIS의 Map Series 아이디어를 코드로 직접 흉내 냅니다. 카운티 목록을 하나씩 꺼내고, 해당 카운티의 트랙만 골라 지도에 칠한 뒤, PDF 인쇄기에 한 페이지씩 쌓아 118페이지 atlas를 완성합니다.

Step 1

데이터와 변수 선택

geopandas로 카운티 경계와 CalEnviroScreen 4.0 트랙 shapefile을 불러옵니다. 그다음 PolBurdP와 AsthmaP 두 변수를 atlas의 주제 지표로 고정합니다.

Step 2

County Loop와 자동 필터링

정렬된 58개 카운티 목록을 반복문으로 돌면서, 매번 ces_tracts['County'] == county_name 조건으로 해당 카운티의 센서스 트랙만 골라냅니다.

Step 3

고정 범례와 동적 페이지 번호

vmin=0, vmax=100으로 모든 지도 범례를 같은 0-100 percentile 척도에 묶습니다. 제목과 footer는 코드가 매 페이지마다 새로 찍어 넣습니다.

👨‍💻 Python 코어 로직 해설

※ 지식 창조자의 무기: 실제 `unit04_automation_pipeline.py`가 하는 일을 한 줄씩 풀어 쓴 설명입니다. ArcGIS의 Map Series 기계를 파이썬으로 직접 만든다고 생각하면 됩니다.

[핵심 로직 1] 재료 창고 열기: 카운티와 CalEnviroScreen 데이터 불러오기
import geopandas as gpd # 카운티 경계 파일을 엽니다. 이 파일은 Map Series의 '인덱스 레이어' 역할을 합니다. # 쉽게 말해, 지도책의 목차 카드 58장을 가져오는 단계입니다. counties = gpd.read_file(self.county_path) # CalEnviroScreen 4.0 센서스 트랙 shapefile을 엽니다. # 이 안에는 오염 부담, 천식, 인구 취약성 같은 실제 색칠 재료가 들어 있습니다. ces_tracts = gpd.read_file(self.ces_path)
[핵심 로직 2] 두 가지 색칠 주제 정하기
# 과제 조건: CalEnviroScreen 데이터 안에서 관련 변수 2개를 골라야 합니다. # 여기서는 환경 부담을 보여주는 PolBurdP와 건강 결과를 보여주는 AsthmaP를 고릅니다. var1 = "PolBurdP" var1_label = "Pollution Burden Percentile" var2 = "AsthmaP" var2_label = "Asthma Percentile" # 두 변수를 한 바구니에 담아두면, 같은 반복문으로 2종류 지도를 찍어낼 수 있습니다. variables = [(var1, var1_label), (var2, var2_label)]
[핵심 로직 3] 58개 카운티를 순서대로 넘기는 지도책 컨베이어 벨트
# 카운티 이름을 알파벳 순서로 정렬합니다. # 그리고 reset_index를 해서 페이지 번호가 중간에 튀지 않도록 새 번호표를 붙입니다. counties = counties.sort_values("name").reset_index(drop=True) # 58개 카운티 × 2개 변수 = 지도 페이지 116장. # 표지와 요약 페이지까지 합치면 PDF 전체는 118페이지입니다. total_counties = len(counties) map_page_count = total_counties * len(variables) total_pdf_pages = map_page_count + 2
[핵심 로직 4] PDF 인쇄기 열고, 카운티별 지도 두 장씩 쌓기
from matplotlib.backends.backend_pdf import PdfPages # PdfPages는 여러 장짜리 PDF를 만드는 인쇄기입니다. # with 안에 있는 동안에는 계속 종이를 넣을 수 있고, 블록이 끝나면 PDF가 완성됩니다. with PdfPages(self.output_file) as pdf: map_index = 0 # 카운티 카드를 한 장씩 꺼냅니다. Alameda, Alpine, Amador ... 이런 순서입니다. for county_index, row in counties.iterrows(): county_name = row["name"] # CalEnviroScreen 전체 트랙 중에서 현재 카운티에 속한 트랙만 골라냅니다. county_tracts = ces_tracts[ces_tracts["County"] == county_name] # 같은 카운티에 대해 변수 2개를 차례로 그립니다. # 그래서 한 카운티당 Pollution map 1장, Asthma map 1장이 나옵니다. for column, label in variables: page_num = map_index + 3 # 1쪽 표지, 2쪽 요약 다음부터 지도 시작 self._create_county_map(pdf, county_tracts, county_name, column, label, page_num, ...) map_index += 1
[핵심 로직 5] 한 장의 지도 페이지를 그리는 실제 도장
# column은 이번에 색칠할 지표입니다. 예: PolBurdP 또는 AsthmaP. # vmin=0, vmax=100은 모든 페이지의 색상 자를 똑같이 맞추는 약속입니다. data.plot( column=column, cmap="YlOrRd", legend=True, ax=ax, edgecolor="#ffffff", linewidth=0.15, legend_kwds={"label": label, "orientation": "horizontal"}, vmin=0, vmax=100, ) # ArcGIS Dynamic Text처럼 제목과 페이지 번호를 매번 새로 찍습니다. # 사람은 제목을 116번 바꾸지 않고, 코드가 알아서 도장을 찍습니다. ax.set_title(f"{county_name} County: {label}", fontsize=16, pad=20) plt.figtext(0.5, 0.055, f"PDF Page {page_num} of {total_pdf_pages} | Map Page {map_page_num} of {total_map_pages}") # 마지막으로 방금 그린 한 장을 PDF 인쇄기에 넣습니다. pdf.savefig(fig) plt.close()
