Unit 02: 네트워크 분석 및 상권 평가 (Network Analysis & Market Research)

🌏 1. 기본 개념 (공통 목표)

  • 📌 해결 과제: 단순히 직선거리가 아니라, 사람들이 이동하는 '도로망'을 따라 이동 시간과 접근성을 분석합니다. 이를 통해 스타벅스 매장의 서비스 영역(상권)을 평가하고, 상권 내 인구수를 집계하여 폐점 여부를 결정합니다.
  • 📌 필요한 데이터:
       - 도로 네트워크: 교차로(Node)와 도로(Edge) 정보가 담긴 데이터.
       - 시설 위치: 분석의 기점이 되는 스타벅스 매장 위치(Point).
       - 인구 통계: 상권 내 인구수를 계산하기 위한 행정 구역(Polygon) 데이터.
  • 📌 핵심 공간 개념:
       - 네트워크 위상: 도로와 교차로가 어떻게 서로 연결되어 있는지(연결성 규칙)에 대한 구조.
       - 다익스트라 알고리즘: 미로에서 가장 빠른 길을 찾는 것처럼, 목적지까지 최단 시간 경로를 찾는 탐색 방법.
       - 등시선 (Isochrones): "출발지에서 차로 3분 안에 도착할 수 있는 모든 곳"을 이어 만든 서비스 구역.

ArcGIS Pro 구현

도구 숙련자 및 오퍼레이터 양성

ArcGIS 워크플로우는 Network Analyst 기능을 사용하여 도로 데이터를 '살아있는 네트워크'로 만드는 설정에 집중합니다.

Step 1

네트워크 데이터셋 구축

단순한 선 데이터를 길 찾기가 가능한 도로 데이터로 변환합니다. 일방통행, 제한 속도 등을 마법사 창에서 일일이 설정합니다.

Step 2

서비스 지역 솔버 (Solver)

스타벅스 위치를 등록하고 '3분'이라는 시간 한계를 입력한 뒤, 도로를 따라 퍼져나가는 등시선을 생성합니다.

Step 3

공간 결합 (Spatial Join)

상권 영역과 인구 통계 블록을 포개어, 상권 내에 거주하는 고객수를 테이블로 합산해 냅니다.

Python 로직 및 아키텍처

지식 창조자 및 GIS 엔지니어 양성

파이썬 방식은 그래프 이론을 바탕으로 도로망을 점과 선의 수학적 모델로 변환하여 메모리에서 직접 계산합니다. 이번 실행에서는 캐시된 Riverside 운전 도로망, Starbucks 점 자료, ACS block group 자료를 사용해 3분 상권과 폐점 후보, 대체 매장 이동시간을 다시 계산했습니다.

Step 1

오픈 소스 그래프 생성 (OSMnx)

OSMnx 라이브러리를 사용하여 OpenStreetMap에서 도시의 모든 도로와 교차로를 수학적 '그래프' 구조로 불러옵니다.

Step 2

알고리즘적 경로 탐색 (NetworkX)

표준 라이브러리인 networkx의 다익스트라 알고리즘을 호출하여 이동 시간이 3분 이내인 모든 지점을 탐색합니다.

Step 3

메모리 기반 상권 생성 및 집계

Shapely로 점들을 감싸는 다각형을 만들고 GeoPandas로 인구수를 즉시 집계하는 파이프라인을 설계합니다.

👨‍💻 Python 코어 로직 해설

※ 지식 창조자의 무기: 유료 도구 없이 수학으로 상권을 직접 설계하는 법.

[핵심 로직 1] OSMnx를 이용한 도로망 그래프 구축
import osmnx as ox # 1. 특정 지역(리버사이드)의 도로망 데이터를 그래프 형태로 가져옵니다. G = ox.graph_from_place("Riverside, CA, USA", network_type="drive") # 2. 도로의 제한 속도를 바탕으로 각 도로를 지나는 '시간'을 계산하여 가중치로 넣습니다. G = ox.add_edge_speeds(G) G = ox.add_edge_travel_times(G)
[핵심 로직 2] 다익스트라 알고리즘 기반 상권 탐색
import networkx as nx # 1. 'ego_graph' 명령어로 특정 스타벅스에서 3분 이내의 도로망만 뽑아냅니다. sub_g = nx.ego_graph(G, center_node, radius=180, distance='travel_time') # 2. 찾아낸 점들을 하나로 모아 'convex_hull' 알고리즘으로 감싸 상권을 그립니다. from shapely.geometry import MultiPoint service_area = MultiPoint(node_points).convex_hull
[핵심 로직 3] 인구 통계 공간 결합 (Spatial Join)
import geopandas as gpd # 1. 만들어진 3분 상권 영역(Polygon) 안에 미국 인구 조사(ACS) 통계 블록의 중심점을 포갭니다. joined = gpd.sjoin(bg_centroids, isochrones, how="inner", predicate="within") # 2. 각 스타벅스 매장(Store_ID)별로 포개어진 통계 블록의 총 인구수(Total_Pop)를 모두 합산합니다. pop_summary = joined.groupby('Store_ID')['Total_Pop'].sum()
[핵심 로직 4] 목적지 비용 매트릭스 (OD Matrix) 생성
# 폐점이 결정된 매장에서 다른 모든 스타벅스 지점까지 도로를 따라 걸리는 시간을 구합니다. lengths = nx.single_source_dijkstra_path_length(G, closed_node, weight='travel_time') # 결과를 분 단위로 환산하고 가장 가까운 대체 매장 순으로 정렬합니다. travel_time_mins = lengths[dest_node] / 60.0
