Unit 05: 원격 데이터 서비스 기반 재해 지도 제작

🌐 1. 기본 개념 (공통 목표)

  • 📌 해결 과제: 미국의 한 지역을 골라 flood 또는 geologic hazard 지도를 만들되, 모든 레이어를 local shapefile이 아니라 remote data service에서 직접 불러와야 합니다.
  • 📌 이번 Python 산출물의 시나리오: Oakland 시 비상관리/도시계획 담당자가 원격 서비스만 사용해 지진성 지반재해 위험을 빠르게 파악할 수 있는 정적 지도를 요청했습니다.
  • 📌 사용한 원격 서비스:
       - USGS NAIP ImageServer: 원격 imagery 배경.
       - California Conservation / CGS fault traces: 과제 링크 계열의 지진 단층 서비스.
       - California GIS liquefaction service: 액상화 위험 구역.
       - California Quaternary Faults FeatureServer: 학생이 직접 고른 추가 원격 hazard source.
  • 📌 핵심 공간 개념:
       - Remote service: 파일을 내려받지 않고 지도 서버가 필요한 범위만 그때그때 보내주는 방식.
       - MapServer/ImageServer/FeatureServer: ArcGIS Server가 지도 이미지, 래스터 이미지, 벡터 피처를 내보내는 대표 서비스 형식.
       - Service manifest: 어떤 원격 URL을 썼는지, 열렸는지, 몇 개 피처를 받았는지 남기는 출처 영수증.

ArcGIS Pro 구현

도구 숙련자 및 오퍼레이터 양성

ArcGIS 방식은 로컬 파일을 Add Data로 추가하는 것이 아니라, Catalog에서 ArcGIS Server 또는 WMS/REST endpoint에 연결한 뒤 서비스 레이어를 지도에 올리는 과정입니다.

Step 1

원격 서버 연결

Catalog pane에서 ArcGIS Server connection을 만들고, course tip에 있는 USGS NAIP 또는 California Conservation 서버 URL을 등록합니다.

Step 2

재해 레이어 3개 이상 추가

지도에는 최소 3개 원격 서비스가 동시에 들어가야 합니다. 한 개는 과제 링크에서, 한 개는 직접 찾은 외부 서비스에서 가져와야 합니다.

Step 3

정적 지도와 Project ZIP 제출

Layout에서 지도, 범례, 출처, 제목을 정리해 PDF로 내보냅니다. ArcGIS 과제 제출물은 PDF map과 APRX project directory ZIP입니다.

Python 로직 및 아키텍처

지식 창조자 및 GIS 엔지니어 양성

Python 방식은 ArcGIS Pro의 “서버 연결 후 레이어 추가”를 코드로 재현합니다. 서버 주소를 목록으로 만들고, 필요한 bbox만 요청하고, 받은 이미지와 GeoJSON을 겹쳐 지도와 PDF를 자동 생성합니다.

Step 1

서비스 주소와 BBOX 정의

분석 범위를 Oakland 주변 네 모서리 좌표로 정하고, 각 원격 서비스 URL을 manifest에 남깁니다.

Step 2

REST 요청으로 자료 받기

requests.get()이 GIS 서버에 “이 범위의 imagery와 hazard feature를 보내달라”고 요청합니다.

Step 3

지도·표·PDF 자동 생성

GeoPandas로 원격 GeoJSON을 읽고, Matplotlib로 정적 지도 PNG를 그린 뒤, ReportLab으로 PDF 보고서를 만듭니다.

👨‍💻 Python 코어 로직 해설

※ 지식 창조자의 무기: 실제 `unit05_automation_pipeline.py`가 수행한 원격 서비스 기반 hazard mapping 흐름입니다. 주석은 초보자도 따라올 수 있게 풀어 썼습니다.

[핵심 로직 0] 원격 서비스 주소 목록 만들기
# RemoteService는 자료 출처 영수증입니다. # 서비스 이름, URL, 과제 링크 여부, 역할을 한 줄 카드처럼 보관합니다. RemoteService( key="naip", label="USGS NAIP Imagery", url="https://imagery.nationalmap.gov/arcgis/rest/services/USGSNAIPImagery/ImageServer", course_linked="Yes", role="Remote imagery basemap/context", ) # BBOX는 지도 창의 네 모서리입니다. # 왼쪽, 아래, 오른쪽, 위 순서로 Oakland 주변 범위를 잡습니다. BBOX = (-122.36, 37.70, -122.10, 37.90)
[핵심 로직 1] 원격 imagery를 그림 바탕으로 가져오기
# exportImage는 ImageServer에게 "이 네모 범위만 그림으로 잘라 보내줘"라고 말하는 요청입니다. # 사람이 ArcGIS에서 NAIP 레이어를 켜는 일을 코드가 대신합니다. params = { "bbox": f"{xmin},{ymin},{xmax},{ymax}", "bboxSR": "4326", "imageSR": "4326", "size": "1600,1200", "format": "png", "f": "image", } response = request_url(f"{service.url}/exportImage", params=params) # 서버가 보내준 PNG 이미지를 저장합니다. # 이 파일은 지도 바닥에 깔릴 원격 항공사진입니다. OUTPUT_NAIP.write_bytes(response.content)
[핵심 로직 2] 원격 hazard layer를 GeoJSON으로 요청하기
# query는 FeatureServer/MapServer에게 "이 네모 안에 닿는 도형만 보내줘"라고 묻는 명령입니다. # f=geojson은 Python이 읽기 쉬운 지도 문서 봉투로 달라는 뜻입니다. params = { "where": "1=1", "geometry": f"{xmin},{ymin},{xmax},{ymax}", "geometryType": "esriGeometryEnvelope", "returnGeometry": "true", "outSR": "4326", "f": "geojson", } payload = request_url(f"{service.url}/query", params=params).json() # GeoPandas가 원격 서버에서 온 feature 봉투를 지도 도형 표로 바꿉니다. gdf = gpd.GeoDataFrame.from_features(payload["features"], crs="EPSG:4326")
[핵심 로직 3] Oakland 범위 안에서 위험 면적과 단층 길이 계산하기
# WGS84 위경도는 길이/면적 계산에 불편합니다. # 그래서 California Albers 좌표계(EPSG:3310)로 바꾼 뒤 km, sq km를 계산합니다. projected = liquefaction.to_crs("EPSG:3310") liquefaction_area_km2 = projected.area.sum() / 1_000_000 fault_length_km = ap_faults.to_crs("EPSG:3310").length.sum() / 1000 # 이번 실행 결과: 지도 창의 23.3%가 liquefaction zone과 겹쳤습니다.
[핵심 로직 4] NAIP 위에 hazard layer를 겹쳐 static map 만들기
# 먼저 원격 NAIP 이미지를 지도 바닥에 깝니다. ax.imshow(image, extent=[xmin, xmax, ymin, ymax], origin="upper") # 그 위에 액상화 구역은 노란 반투명 폴리곤으로 칠합니다. liquefaction.plot(ax=ax, facecolor="#f4d03f", alpha=0.45) # 단층선은 빨간색과 보라색 선으로 올려서 위험 축이 눈에 보이게 합니다. ap_faults.plot(ax=ax, color="#e74c3c", linewidth=2.1) q_faults.plot(ax=ax, color="#8e44ad", linewidth=1.6) # 마지막으로 PNG 지도로 저장합니다. fig.savefig(OUTPUT_MAP, dpi=220)
[핵심 로직 5] PDF 보고서와 manifest를 인쇄하듯 저장하기
# manifest는 "어떤 원격 서비스를 썼는지" 남기는 출처 영수증입니다. # 과제의 핵심 조건인 remote service 사용 여부를 증명합니다. manifest.to_csv(OUTPUT_MANIFEST, index=False) # ReportLab은 PDF 인쇄기입니다. # 지도 그림, 요약 표, 원격 서비스 ledger를 차례대로 종이에 올립니다. doc = SimpleDocTemplate(str(OUTPUT_PDF), pagesize=letter) elements.append(Image(str(OUTPUT_MAP), width=510, height=382)) elements.append(summary_table) elements.append(manifest_table) doc.build(elements)
